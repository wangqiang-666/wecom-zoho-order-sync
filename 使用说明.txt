================================================================
企微表格 → ZOHO 订单 自动同步 使用说明
================================================================

【一句话】
在企微智能表格录入订单 → 把"是否确定导入"改为"导入" → 系统自动创建 ZOHO 订单。（！！！当你录完一条销售单后，最后的步骤：是否确定导入=》导入）

================================================================
一、录入流程
================================================================

1. 在企微智能表格对应子表（CS_Rose / CS_Nick 等）填写一行订单
2. 必填字段填全（主题、渠道名称等）=》另外后台配置网址页面：  可以进行配置检测必须要填入的字段去约束你的填入字段不能为空，也就是提醒你不能为空
3. 把"是否确定导入"列改为"导入"
4. 等10秒左右，可以"导入状态"列结果

================================================================
二、状态说明（"导入状态"列只有三种）
================================================================

  导入中             —— 系统已接到，正在处理，等几秒
  导入成功 (zoho=xxx) —— 已创建 ZOHO 订单，xxx 是 ZOHO 记录 ID
  导入失败: 原因      —— 失败，按"原因"修改后重试

================================================================
三、铁律（必须知道）
================================================================

★ 一行企微记录 = 一条 ZOHO 订单 ★

★ 已同步成功的行（"导入状态"显示"导入成功"），系统不会再动 ZOHO ★
   - 想改 ZOHO 订单内容 → 直接去 ZOHO 改，不要改企微
   - 在已成功的行上改内容 → 系统会当作"全新订单"再创建一条新 ZOHO 记录
     旧的 ZOHO 记录保持不变

★ 想新建一条 ZOHO 订单：
   方法 A：在企微表格新增一行，填内容，点"导入"
   方法 B：删除旧行后在同位置重新录入，点"导入"
   两种方式都会创建独立的新 ZOHO 记录

================================================================
四、失败重试
================================================================

【真失败】"导入失败: xxx"
   → 按提示修改企微行的内容（比如补填渠道名称） → 系统自动重试
   → 或：清空"导入状态"列 → 系统会重试一次

【假失败】ZOHO 已经创建成功，但企微状态显示异常
   → 不用管，下一轮扫描会自动补上"导入成功"
   → 绝不会重复创建 ZOHO 记录

================================================================
五、批量录入
================================================================

- 支持一次粘贴 50+ 行后批量点"导入"
- 系统并发处理（4 worker），50 行约 30-60 秒处理完
- 同一批次内订单确认编号重复 → 全部标失败，提示用户改号

================================================================
六、常见问题
================================================================

Q: 点了"导入"很久没反应？
A: 看"导入状态"列。空白超过 30 秒检查：
   1. 服务是否在跑（联系管理员）
   2. 是否填了"是否确定导入"=导入

Q: 改了内容为什么显示新建了一条 ZOHO 记录？
A: 这是设计：已同步行改内容 = 新订单。想改原 ZOHO 订单请去 ZOHO 改。

Q: 删了行 ZOHO 里的订单还在吗？
A: 在。删企微行不影响 ZOHO 记录，需要的话去 ZOHO 手动删。

Q: 同号订单可以跨天录入吗？
A: 可以。订单号在不同子表/不同行可以重复使用（跨天复用是常见操作）。
   但同一批次内同号会被识别为冲突。

Q: "是否确定导入"已经填了"导入"，还需要点击吗？
A: 任何修改（包括改回"导入"两字）都会触发企微 webhook 通知系统扫描。

================================================================
七、子表管理
================================================================

- 默认监听 CS_ 开头的所有子表（CS_Rose, CS_Nick, ...）
- 新增子表只要名字以 CS_ 开头会自动纳入
- 黑名单/白名单可在 Web UI 配置

================================================================
八、出问题找谁
================================================================

- 状态列长期空白 / 服务异常 → 联系技术管理员
- 同步内容映射错误（字段对不上）→ 联系技术管理员调整 field-map
- ZOHO 端记录有问题 → 直接去 ZOHO 操作

================================================================
九、日常维护（重要）
================================================================

★ 建议每天下班前删除本子表"导入状态=导入成功"的行 ★

好处：
- 加快同步扫描速度（行少=更快）
- 减少 API 调用压力
- 避免表格越来越长难以查看

安全保证：
- 删企微行 ≠ 删 ZOHO 订单，ZOHO 记录不受影响
- 就算同号订单跨天再录入也不会冲突
- 删行后 7 天内 DB 还保留记录可追溯
