## 特殊字段处理

| 字段 | 处理 |
|------|------|
| **订单所有者** (Owner) | 主表列仅作展示；同步用 `ZOHO_DEFAULT_OWNER_EMAIL` 反查 user_id 固定传入 |
| **主题** (Name) | 为空时自动填充 `订单确认编号` 或 `订单-{时间戳}` |
| **订单确认编号** (field235) | 只承载真实业务编号；若主表为空，后端不再用企微 `record_id` 回填，避免干扰 ZOHO 侧文件编号生成 |
| **币种** (field22) | 主表没这列，从 `.env` 的 `ZOHO_DEFAULT_CURRENCY` 取（默认 RMB） |
| **人民币快递金额** (field218) | 主表没这列，自动赋值为 `field20`（原币订单金额）的值 |
| **订单状态** (field62) | 主表没这列，插入时默认赋值为 `"正常"` |
| **渠道名称 / 业务细类** | lookup 类型：录单员填显示名，服务查对应 ZOHO 模块拿 record_id；**待补模块 API name** |

## 值转换规则

| 类型 | 规则 |
|------|------|
| text / textarea | 原样字符串，去首尾空格 |
| integer         | 允许带千分位逗号，必须为整数 |
| double          | 允许千分位、¥￥$ 符号 |
| boolean         | `是/true/1/y/yes/√` → true；`否/false/0/n/no/×` → false |
| date            | 支持 `2026/04/21`、`2026-04-21`、`2026年4月21日`，输出 `YYYY-MM-DD` |
| picklist        | 必须严格匹配白名单 |
| multiselectpicklist | `;|；,，\n` 分隔；每项匹配白名单 |

## 漂移检测

运行 `npm run pull-meta` 会：
1. 拉 ZOHO 最新字段元数据到 `docs/zoho-fields-dump.json`
2. 和 `config/field-map.json` 比对并打印差异（字段删除 / picklist 新增或消失）

建议每次 ZOHO 改配置后都跑一次。
